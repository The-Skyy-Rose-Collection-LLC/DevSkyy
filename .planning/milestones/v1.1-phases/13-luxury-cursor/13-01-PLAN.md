**Part A: Update luxury-cursor.js to detect modal open/close and pause the animation loop.**

Add a modal-awareness system after the existing `mouseenter`/`mouseleave` handlers (before the closing IIFE paren). The approach:

1. **Define a `paused` flag** (initially false) and a stored `rafId` variable to track the animation frame.

2. **Refactor the animation loop**: Change `requestAnimationFrame(animate)` at line 71 to store the returned ID: `rafId = requestAnimationFrame(animate)`. Inside `animate()`, also store the recursive call. This allows `cancelAnimationFrame(rafId)` when pausing.

3. **Create `pauseCursor()` and `resumeCursor()` functions**:
   - `pauseCursor()`: Sets `paused = true`, cancels the animation frame via `cancelAnimationFrame(rafId)`, adds `luxury-cursor-paused` class to `document.body`.
   - `resumeCursor()`: Sets `paused = false`, removes `luxury-cursor-paused` class from `document.body`, restarts the animation loop via `rafId = requestAnimationFrame(animate)`.

4. **Detect modals via MutationObserver**: Create a MutationObserver on `document.body` that watches for attribute changes and child list changes (subtree: true). On each mutation batch, run a check function `checkModals()` that:
   - Queries for any open modal/overlay/dialog using this selector: `[role="dialog"]:not([aria-hidden="true"]):not([inert]), .search-overlay.open, .mobile-menu__panel.open, .col-modal-ov.active, .size-guide-modal.active, .sr-exit-overlay.active, .sr-exit-overlay.visible`
   - If any element matches AND `paused` is false -> call `pauseCursor()`
   - If NO element matches AND `paused` is true -> call `resumeCursor()`

5. **Run `checkModals()` once on init** (after observer setup) to handle edge case where modal is already open when cursor JS loads.

6. **Also listen for the `keydown` Escape event** as a fast-path: when Escape is pressed and `paused` is true, schedule `checkModals()` on next frame (modals close synchronously on Escape, so the DOM state will be updated by next frame).

Important constraints:
- Keep the existing IIFE wrapper and `'use strict'`
