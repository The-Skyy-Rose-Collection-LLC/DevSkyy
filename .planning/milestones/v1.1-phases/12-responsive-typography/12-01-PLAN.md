  <action>
Audit and fix every hardcoded padding/width that can cause horizontal overflow at 320px. The strategy: add `@media (max-width: 480px)` rules (or strengthen existing ones) to cap padding, and fix any fixed-width elements.

**main.css:**
- `.sr-container` already has `padding: 0 20px` at 768px. Add 320px-safe rule: no change needed if 20px works, but verify.

**collection-v4.css — the biggest offender:**
- `.col-hero__content` has `padding: 0 64px 80px` at desktop. The 600px breakpoint sets `padding: 0 20px 48px` and the 480px breakpoint sets `padding: 0 16px 36px`. Verify 16px is safe at 320px. This looks OK.
- `.col-manifesto .col-mf__text` has `padding: 80px 64px`. The 600px breakpoint reduces to `48px 20px` and 480px to `36px 16px`. OK.
- `.col-featured` has `padding: 120px 64px`. The 600px breakpoint reduces to `80px 20px`. OK.
- `.col-catalog` has `padding: 100px 64px`. The 600px breakpoint reduces to `60px 16px` and 480px to `48px 12px`. OK.
- `.col-crossnav` has `padding: 48px 64px`. The 600px breakpoint reduces to `32px 20px` and 480px to `24px 16px`. OK.
- `.col-nl` has `padding: 120px 48px`. The 600px breakpoint reduces to `80px 16px` and 480px to `60px 12px`. OK.
- **ISSUE**: `.col-toolbar` at 600px has `padding: 12px 14px` — OK for 320px.
- **ISSUE**: `.col-hero__meta` uses `gap: 32px` with multiple items that could overflow. At 600px it wraps with `gap: 16px` — OK.
- **ISSUE**: `.col-hero__ctas .col-btn` has `padding: 18px 40px` with `letter-spacing: 6px`. At 600px CTAs stack vertically. But the button text + padding could overflow 320px if the text is long. Add `max-width: 100%` and `box-sizing: border-box` to `.col-btn` to prevent overflow.
- Add to the 480px breakpoint: `.col-btn { padding: 16px 24px; letter-spacing: 4px; }` to shrink buttons for narrow screens.

**homepage-v2.css:**
- `.nav` has `padding: 20px 48px`. No mobile breakpoint exists for 320px. Add: `@media (max-width: 480px) { .homepage-v2 .nav { padding: 16px 16px; } .homepage-v2 .nav.scrolled { padding: 12px 16px; } }`
- `.story` has `padding: clamp(100px, 12vw, 180px) 48px`. Add 480px breakpoint: `.homepage-v2 .story { padding: 60px 16px; }`
- `.quote-section` has `padding: clamp(80px, 10vw, 140px) 48px`. Add: `.homepage-v2 .quote-section { padding: 60px 16px; }`
- `.craft` has `padding: clamp(80px, 10vw, 140px) 48px`. Add: `.homepage-v2 .craft { padding: 60px 16px; }`
- `.newsletter` has `padding: clamp(80px, 10vw, 140px) 48px`. Add: `.homepage-v2 .newsletter { padding: 60px 16px; }`
- `.ft` (footer) has `padding: 80px 48px 40px`. Add: `.homepage-v2 .ft { padding: 48px 16px 24px; }`
- `.press` has `padding: 40px 48px`. Add: `.homepage-v2 .press { padding: 24px 16px; }`
- `.hero-title` has `white-space: nowrap` at desktop. The 768px breakpoint sets `white-space: normal`. Verify this is fine at 320px. (It is.)
- `.col-header` has `padding: 0 48px`. Add to 768px rule: `.homepage-v2 .col-header { padding: 0 16px; }`
- `.lookbook-header` has `padding: 0 48px`. Add: `.homepage-v2 .lookbook-header { padding: 0 16px; }`
- Add all these in a new `@media (max-width: 480px)` block at the end of the RESPONSIVE section in homepage-v2.css.

**front-page.css:**
- `.hero__content` has `padding: 2rem` — fine at 320px.
- `.collections__grid` has `grid-template-columns: repeat(auto-fit, minmax(320px, 1fr))`. At 320px viewport with padding, the minmax(320px) could force overflow. Change to `minmax(min(320px, 100%), 1fr)` or use the existing 768px rule that sets `grid-template-columns: 1fr`.
- `.testimonials__grid` has `minmax(320px, 1fr)` — same issue. At 768px it switches to `1fr`. But between 480-768px this could overflow. Fix: `minmax(min(300px, 100%), 1fr)`.

**footer.css:**
- `.footer-newsletter__input` has `width: 260px`. At 768px the container stacks and input gets `width: 100%`. OK. But verify the input-group doesn't overflow at 320px when stacked. The `min-width: 200px` is fine.
- `.footer-grid__container` at 480px uses `grid-template-columns: 1fr`. OK.
- `.footer-copyright__container` at 480px stacks. OK.

**General safeguard**: Add to main.css body rule: verify `overflow-x: hidden` is already set (it is). Also ensure `.sr-container` doesn't exceed viewport — already has `max-width: 1400px; padding: 0 20px` at mobile. OK.

**main.css:**
- `.sr-back-to-top` at 480px: currently `width: 42px; height: 42px`. Change to `width: 44px; height: 44px`.

**header.css:**
- `.navbar__action-btn`: currently 40x40. Change to `min-width: 44px; min-height: 44px; width: 44px; height: 44px;`
- `.navbar__hamburger`: currently 40x40. Change to `min-width: 44px; min-height: 44px; width: 44px; height: 44px;`
- `.mobile-menu__close`: currently 36x36. Change to `width: 44px; height: 44px;`
- `.mobile-menu__list > li > a, .mobile-menu__nav > ul > li > a`: currently `padding: 0.875rem 1.5rem`. This gives adequate height (~44px+) due to font-size + padding. Verify: 0.8125rem (13px) + 2*0.875rem (28px) = 41px. Bump padding to `1rem 1.5rem` for 45px minimum.
- `.mobile-menu__list .sub-menu a, .mobile-menu__nav ul .sub-menu a`: currently `padding: 0.6rem 1.5rem 0.6rem 2.5rem`. Height = 14.4px + 2*9.6px = 33.6px. Too small. Change to `padding: 0.75rem 1.5rem 0.75rem 2.5rem` for ~38px, then add `min-height: 44px; display: flex; align-items: center;`.
- `.mobile-menu__link`: currently `padding: 0.75rem 1.5rem`. Height = 14px + 24px = 38px. Add `min-height: 44px;`.

**footer.css:**

- `.footer-copyright__menu a`: text links in footer nav. Add `min-height: 44px; display: inline-flex; align-items: center;` to give adequate tap area.
- `.footer-newsletter__submit`: has `padding: 0.75rem 1.25rem`. Height ~ 36px. Add `min-height: 44px;`.

**collection-v4.css:**
- `.col-card__heart`: currently 36x36. Change to `width: 44px; height: 44px;`. Adjust `top: 50px` to `top: 46px` and `right: 14px` to `right: 10px` to keep visual position.

- `.col-modal__sz`: currently `min-width: 46px; height: 46px`. Already >= 44px. OK.
- `.col-modal__close`: currently 40x40. Change to `width: 44px; height: 44px;`.
- `.col-card__view-btn`: currently `padding: 10px 18px`. Height ~ 31px. Change to `padding: 12px 18px; min-height: 44px;`.
