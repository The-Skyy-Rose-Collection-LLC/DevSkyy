<task type="auto">
  <name>Task 1: Create idempotent branch protection setup script</name>
  <files>scripts/setup-branch-protection.sh</files>
  <action>
Create `scripts/setup-branch-protection.sh` with `set -euo pipefail` and the following behavior:

**Constants:**
- OWNER="The-Skyy-Rose-Collection-LLC"
- REPO="DevSkyy"
- BRANCH="main"

**Step 1 -- Enable auto-merge on the repository:**
```
gh api --method PATCH "repos/$OWNER/$REPO" -f allow_auto_merge=true
```

**Step 2 -- Set branch protection rules via PUT:**
Use `gh api --method PUT "repos/$OWNER/$REPO/branches/$BRANCH/protection" --input -` with a heredoc JSON payload containing:
- `required_status_checks.strict`: true (PR-02: branch must be up-to-date)
- `required_status_checks.checks`: array of 4 objects, each with `app_id: -1`:
  1. context: "🔍 Lint & Static Analysis"
  2. context: "🐍 Python Tests"
  3. context: "🔐 Security Scan"
  4. context: "⚛️ Frontend Tests"
- `enforce_admins`: true (block admin bypass, block direct pushes)
- `required_pull_request_reviews`: null (no human reviewers required -- CI is the reviewer)
- `restrictions`: null (no user/team restrictions)
- `allow_force_pushes`: false (block force pushes to main)
- `allow_deletions`: false

**Step 3 -- Verify by reading back protection:**
Query the protection endpoint and verify with `--jq`:
1. Check count equals 4: `.required_status_checks.checks | length`
2. Strict mode is true: `.required_status_checks.strict`
3. Enforce admins is true: `.enforce_admins.enabled`
4. Force pushes blocked: `.allow_force_pushes.enabled` is false
5. Auto-merge enabled: query `repos/$OWNER/$REPO --jq '.allow_auto_merge'` is true

Print clear PASS/FAIL for each check. Exit 1 if any check fails, exit 0 if all pass.

**Support a `--verify` flag** that skips Steps 1-2 and only runs Step 3 (read-only verification).

Make the script executable (chmod +x). Use per user decision: classic branch protection rules (not rulesets), `gh api` for automation (no manual GitHub UI steps).
