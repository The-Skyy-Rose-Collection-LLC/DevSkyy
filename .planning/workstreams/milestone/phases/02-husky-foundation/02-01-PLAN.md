**Step 1: Initialize Husky v9 at monorepo root.**

Run `npx husky init` from the repo root. This will:
- Create `.husky/` directory
- Create `.husky/pre-commit` with default content (`npm test`)
- Set `core.hooksPath` to `.husky/_`
- Overwrite the `prepare` script in `package.json` to just `"husky"`

**Step 2: Fix the prepare script.**

After `npx husky init` overwrites `prepare`, update `package.json` so the prepare script reads:
```json
"prepare": "husky && npm run build"
```
This chains Husky's `core.hooksPath` setup with the existing TypeScript build. Per user decision, this is the exact format to use.

**Step 3: Replace pre-commit hook content.**

Replace the default `npm test` content in `.husky/pre-commit` with a proof-of-life echo. The file should contain exactly:
```bash
echo "pre-commit hook active"
```
No shebang needed (Husky v9 stubs handle execution). No lint/test logic -- Phase 3 owns that.

**Step 4: Create Git LFS hooks in `.husky/`.**

Since `core.hooksPath` redirects Git away from `.git/hooks/`, recreate the 4 LFS hooks. Use `exit 0` (not `exit 2`) because this project doesn't actively track files with LFS (assets are on HuggingFace Hub). The hooks are a safety net.

Create these 4 files, each with the same pattern:

`.husky/post-checkout`:
```bash
command -v git-lfs >/dev/null 2>&1 || exit 0
git lfs post-checkout "$@"
```

`.husky/post-commit`:
```bash
command -v git-lfs >/dev/null 2>&1 || exit 0
git lfs post-commit "$@"
```

`.husky/post-merge`:
```bash
command -v git-lfs >/dev/null 2>&1 || exit 0
git lfs post-merge "$@"
```

`.husky/pre-push`:
```bash
command -v git-lfs >/dev/null 2>&1 || exit 0
git lfs pre-push "$@"
```

Ensure all 5 hook files (pre-commit + 4 LFS) are executable: `chmod +x .husky/pre-commit .husky/post-checkout .husky/post-commit .husky/post-merge .husky/pre-push`

**Step 5: Verify `core.hooksPath` is correct.**

Run `git config core.hooksPath` and confirm it returns `.husky/_`. If it does not, run `npx husky` again or set it manually: `git config core.hooksPath .husky/_`.

**Step 6: Create permanent verification script.**

Create `scripts/verify-hooks.sh` with these checks:
1. `.husky/pre-commit` exists (`test -f`)
2. `.husky/pre-commit` is executable (`test -x`)
3. `core.hooksPath` is set to `.husky/_`
4. A test commit triggers the hook (create a temp branch, stage a temp file, run `git commit --allow-empty -m "hook test"`, capture output for "pre-commit hook active", then clean up: delete the commit with `git reset HEAD~1`, remove temp file, delete temp branch)

The script should:
- Use `set -euo pipefail`
- Print PASS/FAIL for each check
- Exit 0 if all pass, exit 1 if any fail
- Handle cleanup in a trap so temp artifacts are always removed
- Be idempotent and safe to run at any time

Make it executable: `chmod +x scripts/verify-hooks.sh`

**Step 7: Quick smoke test.**

Run `bash scripts/verify-hooks.sh` to confirm all checks pass.
