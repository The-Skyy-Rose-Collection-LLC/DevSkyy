Constants to define at module level:

```
VALID_COLLECTIONS = {"black-rose", "love-hurts", "signature", "kids-capsule"}

SKU_PREFIX_TO_COLLECTION = {
    "br-": "black-rose",
    "lh-": "love-hurts",
    "sg-": "signature",
    "kc-": "kids-capsule",
}

# Only SKUs confirmed present in CSV as pre-order rows.
# Retired SKUs (lh-001, sg-d01, br-d01..d04) are absent from the CSV
# and must NOT appear here — asserting their pre_order flag would KeyError.
PRE_ORDER_SKUS = {"br-004", "br-005", "br-006", "sg-001"}
```

Fixture: `@pytest.fixture(scope="module") def rows(): return read_catalog_rows()`

Three test functions:

1. `test_preorder_skus_flagged(rows)`:
   - Build a dict `{row["sku"]: row for row in rows}` for O(1) lookup.
   - For each sku in PRE_ORDER_SKUS: assert `bool_col(catalog[sku], "pre_order") or bool_col(catalog[sku], "is_preorder")` is True.
   - Failure message: f"SKU {sku} expected to be flagged pre-order in catalog CSV"
   - If a sku from PRE_ORDER_SKUS is missing from the catalog entirely, the test should fail with: f"SKU {sku} not found in catalog — was it retired? Update PRE_ORDER_SKUS."

2. `test_no_cross_collection_sku_leakage(rows)`:
   - For each row in rows (skip rows where sku is empty/None):
     - Determine expected collection from SKU_PREFIX_TO_COLLECTION by checking which prefix the sku starts with.
     - If sku has no known prefix, skip (no assertion — unknown prefixes are not a leakage case).
     - Assert `row["collection"] == expected_collection` with message: f"SKU {row['sku']} has prefix implying {expected_collection} but collection column is {row['collection']!r}"

3. `test_collections_are_valid(rows)`:
   - For each row in rows where `row.get("collection")` is truthy:
     - Assert `row["collection"].strip() in VALID_COLLECTIONS`
     - Failure: f"SKU {row.get('sku', '?')} has unknown collection {row['collection']!r}"

Import block at top:
