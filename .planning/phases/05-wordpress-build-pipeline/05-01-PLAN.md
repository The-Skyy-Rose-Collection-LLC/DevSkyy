**1. Rewrite `webpack.config.js`** (complete replacement):
- Replace the 6 hardcoded entries (3 of which reference non-existent files: main.js, three-init.js, accessibility.js) with `glob.sync('./assets/js/*.js', { ignore: './assets/js/*.min.js' })` dynamic discovery
- Use `const glob = require('glob')` (already available in node_modules)
- Set `output.iife: true` -- these are vanilla browser scripts that set globals like `window.SkyyRose`, NOT ES modules
- Set `output.clean: false` -- output directory IS the source directory
- Set `devtool: 'source-map'` for BUILD-03 (external .map files)

- Keep existing performance hints (500KB warning threshold)

**2. Create `scripts/build-css.js`** (new file):
- Create `wordpress-theme/skyyrose-flagship/scripts/` directory first
- Use `clean-css` programmatic API (NOT CLI) with `glob.sync` to discover all CSS files:
  - Pattern: `assets/css/**/*.css` with ignore `assets/css/**/*.min.css` (catches system/animations.css via double-star)
  - Also include root `style.css`
- For each CSS file: read source, run through `new CleanCSS({ sourceMap: true })`, write `.min.css` and `.min.css.map`
- CRITICAL for root `style.css`: Use `level: { 1: { specialComments: 'all' } }` OR ensure the CleanCSS instance preserves the first `/*!` comment (the WordPress theme header `/* Theme Name: SkyyRose Flagship ... */`). The theme header MUST start with `/*!` for clean-css to preserve it. Check if it does -- if the existing style.css header uses `/*` (no bang), use a two-pass approach: extract the header comment, minify the rest, prepend the header.
- Print summary to stdout: "[build:css] Minified N files" with list of files processed
- Exit with non-zero code if any file fails (do not swallow errors)

**3. Update `package.json` scripts** (targeted edit):
- Change `"build:css"` from the hardcoded cleancss CLI command to `"node scripts/build-css.js"`
- Keep `"build:js": "webpack --mode production"` (already correct)
- Keep `"build": "npm run build:js && npm run build:css"` (already correct)
- Do NOT modify any other fields in package.json

**4. Fix `.gitignore` subdirectory gap** (targeted edit):
- Add `!assets/css/**/*.min.css` after the existing `!assets/css/*.min.css` line

- This ensures `assets/css/system/animations.min.css` is tracked by git
- Do NOT add un-ignore rules for .map files (they should remain gitignored per existing `*.map` rule -- source maps are local-only)
