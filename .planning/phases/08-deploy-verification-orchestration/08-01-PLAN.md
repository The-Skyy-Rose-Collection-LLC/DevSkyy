    **Then create** `scripts/verify-deploy.sh` following the research skeleton (08-RESEARCH.md "Deep Content Verification Script Skeleton"):

    1. Header: `#!/usr/bin/env bash` with `set -euo pipefail`
    2. Path setup: SCRIPT_DIR, PROJECT_ROOT, ENV_FILE
    3. SITE_URL from `WORDPRESS_URL` env var, defaulting to `https://skyyrose.co`
    4. TIMESTAMP for cache-busting
    5. FAILURES and CHECKS counters
    6. Color logging functions (same pattern as deploy-theme.sh: log_info, log_success, log_error)
    7. `parse_args()` supporting `--help`, `--list`, and optional `--url <URL>` override
    8. HEALTH_CHECKS array with pipe-delimited entries: `"name|path|content_marker"`
       - `"Homepage|/|SKYY ROSE"` (brand text from header.php navbar__gradient-text)
       - `"REST API|/index.php?rest_route=/|namespaces"` (WordPress REST API marker, uses `index.php?rest_route=` per CLAUDE.md)
       - `"Collection: Black Rose|/collection-black-rose/|Black Rose"`
       - `"Collection: Love Hurts|/collection-love-hurts/|Love Hurts"`
       - `"Collection: Signature|/collection-signature/|Signature"`
       - `"About|/about/|SkyyRose"`
    9. `verify_page()` function:
       - Takes name, url, marker as args
       - Uses `curl -sSL -w "\n%{http_code}" --connect-timeout 10 --max-time 30 --retry 2 --retry-delay 3`
       - Appends `?_verify=$TIMESTAMP` for cache busting (use `&` if URL already has `?`)
       - Extracts http_code from last line, body from everything else
       - Checks http_code == 200, then greps body for marker (case-insensitive: `grep -qi`)
       - Increments FAILURES on failure, CHECKS always
       - Returns 1 on failure but does NOT exit the script (caller uses `|| true`)

    11. `main()`: loops HEALTH_CHECKS array, splits on `|`, calls `verify_page` with `|| true` to continue on failure, prints summary, exits with `[[ $FAILURES -eq 0 ]]`

    - Do NOT skip content verification -- HTTP 200 alone is insufficient (the whole point of DEPLOY-04)
    - Use `&_verify=` for the REST API URL (already has `?rest_route=`) and `?_verify=` for others
