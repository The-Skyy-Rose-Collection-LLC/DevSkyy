<objective>
Build the .glb cache manager and Tripo digitization wrapper. The cache persists durable .glb meshes to renders/3d/cache/{sku}.glb so digitization never runs twice for the same SKU. The Tripo wrapper calls generate_from_image() and falls back to Meshy when Tripo returns unusable output.

Purpose: Re-digitizing on every retry burns Tripo credits and adds 60-120s latency. D-05 is explicit — digitize once, version-pin, reuse. The cache is the budget governor for the 3D stage.
Output: skyyrose/elite_studio/glb_cache.py with GLBCacheManager and TripoDigitizer.
</objective>

    MODULE LAYOUT:

    1. Imports: dataclasses, pathlib, logging, os, shutil, typing. No heavy third-party imports at module level. TripoClient and MeshyClient imported lazily inside TripoDigitizer methods (avoids circular imports + allows unit testing with mocks).

    2. Module constants:
       - _DEFAULT_CACHE_DIR: Path = Path("renders/3d/cache") — relative to CWD at runtime
       - logger = logging.getLogger(__name__)

    3. GLBCacheManager class:
       - __init__(self, cache_dir: Path | None = None): sets self._cache_dir = cache_dir or _DEFAULT_CACHE_DIR; calls self._cache_dir.mkdir(parents=True, exist_ok=True)
       - exists(self, sku: str) -> bool: returns (self._cache_dir / f"{sku}.glb").is_file()
       - get(self, sku: str) -> Path | None: returns path if exists(), else None
       - put(self, sku: str, source_glb: Path) -> Path:
         - dest = self._cache_dir / f"{sku}.glb"
         - shutil.copy2(source_glb, dest)
         - logger.info("Cached .glb for SKU %s at %s", sku, dest)
         - return dest
       - invalidate(self, sku: str) -> bool:
         - target = self._cache_dir / f"{sku}.glb"
         - if target.exists(): target.unlink(); return True
         - return False

    4. TripoDigitizer class:
       - __init__(self, cache: GLBCacheManager, tripo_api_key: str | None = None, meshy_api_key: str | None = None):
         - self._cache = cache
         - self._tripo_key = tripo_api_key or os.environ.get("TRIPO3D_API_KEY", "")
         - self._meshy_key = meshy_api_key or os.environ.get("MESHY_API_KEY", "")
       - digitize(self, sku: str, flatlay_path: Path, tmp_dir: Path) -> Path:
         - Cache hit path: if self._cache.exists(sku): logger.info("Cache hit for %s", sku); return self._cache.get(sku)
         - Log cache miss: logger.info("Cache miss for %s — calling Tripo", sku)
         - Call _try_tripo(sku, flatlay_path, tmp_dir) → glb_path: Path | None
         - If glb_path is None: logger.warning("Tripo returned None for %s — falling back to Meshy", sku); glb_path = _try_meshy(sku, flatlay_path, tmp_dir)
         - If glb_path is None: raise RuntimeError(f"Both Tripo and Meshy failed to digitize SKU {sku!r}")
         - self._cache.put(sku, glb_path)
         - return self._cache.get(sku)  # always return cache path, not tmp
       - _try_tripo(self, sku: str, flatlay_path: Path, tmp_dir: Path) -> Path | None:
         - Lazy import: from ai_3d.providers.tripo import TripoClient
         - Instantiate TripoClient using self._tripo_key
         - Call client.generate_from_image(str(flatlay_path), str(tmp_dir))
         - On None return or any Exception: log at WARNING level (include exc_info=True on exception), return None
         - On success: return Path(result_path)
       - _try_meshy(self, sku: str, flatlay_path: Path, tmp_dir: Path) -> Path | None:
         - Lazy import: from ai_3d.providers.meshy import MeshyClient
         - Instantiate MeshyClient using self._meshy_key
         - Call client.generate_from_image(str(flatlay_path), str(tmp_dir / f"{sku}-meshy.glb"))
         - On None return or any Exception: log at WARNING level (include exc_info=True on exception), return None
         - On success: return Path(result_path)

    CRITICAL: Do NOT import TripoClient or MeshyClient at module top level. Lazy import inside each _try_* method. This keeps the module importable without API keys in the environment.

    CRITICAL: The sku parameter passed to GLBCacheManager.get/put/exists must already be sanitized (caller's responsibility via sanitize_sku from 15-02). GLBCacheManager does NOT call sanitize_sku — it trusts the caller. Add a docstring note to this effect.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| flatlay_path → Tripo API | User-controlled flatlay path from manifest.json passed to external paid API |
| Tripo API response → local filesystem write | .glb bytes written to cache; never deserialized as Python object |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-03-01 | Tampering | .glb cache path construction | mitigate | Cache path = cache_dir / f"{sku}.glb" where sku is already sanitized by caller (sanitize_sku from 15-02); GLBCacheManager trusts caller contract; no secondary sanitization needed since _SAFE_RE already enforced upstream |
| T-15-03-02 | Information Disclosure | TRIPO3D_API_KEY in logs | mitigate | _try_tripo logs exc_info=True on failure; ensure exception repr from TripoClient does not include key in traceback. Log message strings must not include self._tripo_key; lazy import means key never appears in module-level log output |
| T-15-03-03 | Denial of Service | .glb file unbounded size | accept | TripoClient's existing <=10MB input guard blocks oversized flatlay uploads; .glb output size is bounded by Tripo's mesh complexity caps (~5MB typical); no additional guard needed at cache layer |
| T-15-03-04 | Tampering | Pickle deserialization of .glb | accept | .glb file read as raw bytes only (shutil.copy2 is byte-for-byte copy); never deserialized via pickle, json.load, or eval; format is binary glTF — safe to copy blindly |
| T-15-03-05 | Spoofing | Meshy fallback as secret primary | accept | Meshy is called only after Tripo returns None or raises; call order is deterministic and logged at INFO level; no silent failover to Meshy without a WARNING log entry |
</threat_model>
