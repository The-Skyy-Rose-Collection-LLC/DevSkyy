       - sku: str — sanitized SKU
       - name: str — product name from CSV

       - bundle_dir: Path — path to data/product-bundles/<dirname>/

       - branding_spec: str — col 15 value from CSV

    4. sanitize_sku(raw: str) -> str:
       - Strips whitespace
       - Rejects empty string → ValueError("SKU must not be empty")
       - Rejects if ".." in raw → ValueError("SKU contains path-traversal sequence")
       - Rejects if raw.startswith("/") or raw.startswith("\\") → ValueError("SKU must not be an absolute path")
       - Rejects if not _SKU_SAFE_RE.match(raw) → ValueError(f"SKU contains illegal characters: {raw!r}")
       - Returns raw.strip()

    5. resolve_sku(raw_sku: str, catalog_path: Path | None = None, bundle_root: Path | None = None) -> ResolvedSKU | None:
       - Calls sanitize_sku(raw_sku) first — propagates ValueError on illegal input
       - Uses catalog_path or CATALOG_CSV_PATH; uses bundle_root or BUNDLE_DIR
       - Opens CSV with csv.DictReader
       - Finds row where row["sku"] == sanitized_sku
       - If row not found → raise ValueError(f"SKU {sanitized_sku!r} not found in catalog")
       - If row["garment_type_lock"] == _ACCESSORY_GARMENT_TYPE or row["render_is_accessory"] == "1":
         - logger.info("SKU %s is accessory — skipping (per D-12/GM-06)", sanitized_sku)
         - return None  (caller must check for None and log to failures.json)
       - Constructs bundle_dir = bundle_root / row["dossier_slug"] if dossier_slug column present and non-empty; otherwise falls back to bundle_root / sanitized_sku
       - manifest_path = bundle_dir / "manifest.json"
       - Returns ResolvedSKU(sku=sanitized_sku, name=row["name"], garment_type=row["garment_type_lock"], bundle_dir=bundle_dir, manifest_path=manifest_path, branding_spec=row.get("branding_spec", ""), is_jersey=(sanitized_sku in _JERSEY_SKUS))

    6. verify_tripo_region(tripo_api_key: str | None = None) -> bool:
       - Import TripoClient locally inside this function (avoids hard dep at module import time): from ai_3d.providers.tripo import TripoClient
       - If tripo_api_key is None, read from os.environ.get("TRIPO3D_API_KEY")
       - If key is empty/None → raise EnvironmentError("TRIPO3D_API_KEY not set — cannot verify Tripo region")

       - Call client.get_balance() — this is a free endpoint (per D-08)
       - On success (no exception): logger.info("Tripo .ai region active; credits available"); return True
       - On exception: logger.warning("Tripo region check failed: %s", exc); return False
       - Do NOT suppress exceptions that indicate wrong region vs transient network error — let caller decide

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-15-02-01 | Tampering | sanitize_sku path-traversal | mitigate | Reject any SKU containing ".." or starting with "/" or "\"; enforce _SKU_SAFE_RE allowlist (alphanumeric + hyphen + underscore, max 64 chars) before any path construction |
| T-15-02-02 | Tampering | sanitize_sku shell injection | mitigate | _SKU_SAFE_RE allowlist blocks semicolons, pipes, backticks, spaces, dollar signs — all shell metacharacters are outside [a-zA-Z0-9_-] |

| T-15-02-04 | Information Disclosure | branding_spec from CSV → Gemini prompts | mitigate | branding_spec is a free-text field; sanitize before inserting into prompts in 15-04 (strip embedded instruction patterns); sku_resolver returns raw value, prompt sanitization is 15-04's responsibility |
