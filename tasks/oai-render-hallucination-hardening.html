OAI Render Pipeline — Hallucination Hardening

Generated 2026-06-09 · 4 parallel research lanes (OpenAI official docs, garment-render community techniques, automated QC gating, local pipeline map) · ~35 sources · Confidence: High on QC-gate design, Medium on prompt-level mitigations (image models are probabilistic)

Executive Summary

The pipeline's prompts already carry strong single-image and fidelity language, yet four failure modes survived it in the pilot batch (multi-panel collages, patch drift, mirrored backs, dropped pair garments). Research confirms why: prompt text alone cannot guarantee output compliance — every production pipeline that got bad-render rates down (one documented case: 15% → <3%) did it with template-locked prompt assembly plus an automated post-generation QC gate with judged retries. The cost asymmetry is decisive: a GPT-4o-mini vision judge at detail:"low" costs ~$0.000075/check against a $0.40 render — the entire 79-image batch can be judged for under five cents. The Playwright visual sweep of the completed 79-image run found zero structural defects (no collages, no mirrored backs, no dropped pair garments), so hardening targets the next batch's tail risk plus the subtler fidelity drift (material, colorway, patch interiors) that text prompts cannot police.

3. Automated QC Gate (the load-bearing fix)

Layered gate, cheapest-first. Judge-call economics: GPT-4o-mini at detail:"low" = 85 image tokens flat ≈ $0.000075/check; even GPT-4o high-detail is $0.0038. 240 worst-case checks < $1 against $32 of renders.

LayerCheckCostCatches
1. Deterministic (Pillow+numpy)Valid PNG, expected 1024×1536, white-gutter projection + full-span separator-line detection$0, ~20msCorrupt output, collage/grid panels
2. VLM judge (GPT-4o-mini, detail:"low", strict JSON schema)Single-image, garment present & matches reference, correct view (front/back), patch/text legible & correct, both garments present (pairs)~$0.0001Mirrored backs, dropped pair garments, patch drift, wrong garment
3. Judged retry loopFail → re-render same plan, max 2 retries; then quarantine to renders/oai/_rejected/ with failure tags for human review. Never infinite-loop.≤2× render cost per failureTransient model misses (the dominant case)
4. Runtime budget sentinelAccumulated actual spend (renders + retries + judge) checked before every paid call — closes the known gap where the $50 cap guards the manifest estimate but not retry storms (worst case was ~$189).$0Runaway retry spend

OpenAI's own Image Evals cookbook is the canonical reference for the judge layer: boolean gates fail before any quality scoring, schema enforced with strict:true, a reason field forces chain-of-thought. Deferred as optional future layers (cost/benefit not justified now): FashionCLIP embedding gate (needs torch; VLM judge covers identity), PaddleOCR (VLM reads stylized script better than classical OCR).
