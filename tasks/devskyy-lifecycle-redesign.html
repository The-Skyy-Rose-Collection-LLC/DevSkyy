TL;DR

DevSkyy already has the capability for all five lifecycle stages — it lacks a stage spine that says which agent owns which stage and what hands off between them. The disease is duplication, not absence: imagery generation exists 6 ways, WP-theme build 3 ways, LLM routing 3 ways, agent base classes 2 lineages, QC 2 implementations. Two stages have no agent owner at all (Prototype, customer-Support). The redesign is mostly consolidation + ownership assignment, plus removing ~1 GB of generated artifacts living in git.
