== Installation ==

### Automatic Installation

1. Log in to your WordPress admin panel
2. Navigate to Plugins > Add New
3. Search for "Skyy Rose AI Agents"
4. Click "Install Now" and then "Activate"

### Manual Installation

1. Download the plugin zip file
2. Upload to `/wp-content/plugins/` directory
3. Extract the files
4. Activate the plugin through the 'Plugins' menu in WordPress

### Configuration

1. After activation, navigate to **AI Agents** in your WordPress admin menu
2. Configure your OpenAI API key in **Settings > Integrations** (optional but recommended)
3. Enable/disable agents based on your needs in **Settings > Agents**
4. Set up monitoring intervals and thresholds in **Settings > General**
5. Run your first analysis from the **Dashboard**

== Frequently Asked Questions ==

= Do I need an OpenAI API key? =

While not required, an OpenAI API key significantly enhances the AI capabilities of the agents. Many features work without it, but for the full experience, we recommend obtaining an API key from OpenAI.

= Will this slow down my website? =

No. The agents are designed to run efficiently in the background without impacting your site's front-end performance. Heavy operations are scheduled during low-traffic periods.

= Is it compatible with WooCommerce? =

Yes! The plugin includes specialized optimizations for WooCommerce stores, including product image optimization, inventory analysis, and e-commerce security measures.

= Can I customize which agents run and when? =

Absolutely. You have full control over which agents are enabled, their running intervals, and their specific configurations through the comprehensive settings panel.

= Does it work with caching plugins? =

Yes, the plugin is compatible with all major caching plugins including WP Rocket, W3 Total Cache, and WP Super Cache.

= Is it multisite compatible? =

Yes, the plugin works with WordPress multisite installations and can be configured per-site or network-wide.

= What happens to my data if I deactivate the plugin? =

The plugin stores all data in custom database tables. When deactivated, your data remains intact. Only when you delete the plugin are the tables and data removed.
