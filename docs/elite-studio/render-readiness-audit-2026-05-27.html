1c. Per-venture CLIs

Each Elite Studio venture has its own CLI under skyyrose/elite_studio/ventures/{photo,video,threed}/cli.py. These are domain-specific (e.g. video venture exposes timelapse + motion-blur ops); the cross-venture surface is the top-level skyyrose.elite_studio CLI above.

1d. MCP server

devskyy_mcp.py is a thin client that forwards to a remote DevSkyy REST API (DEVSKYY_API_URL). No local @mcp.tool decorators — the tool surface is over the wire, not in this repo. Treat as a deployment surface, not a local render path.

2a. Findings

  • Dossiers: 100% (32/32). Every catalog SKU has a parsed markdown dossier.
  • sku_folders mapping: 28% (9/32). The other 23 SKUs cannot resolve a co-located per-SKU logo patch through LogoRegistry.image_path() — it raises SkuFolderUnknownError. Every SKU outside Black Rose's first wave is unmapped.
  • Logo placements: 30/32 SKUs have ≥1. sg-011 and sg-012 have ZERO — the renderer would apply no logos to those SKUs. Original-Label Tee variants need a placement record before rendering.
  • Typography: Carried entirely inside dossier brand-mark and typography sections. Verified via get_product_with_dossier() — no separate font-spec file to drift from.

1 23 SKUs missing sku_folders entry in logo-registry.json Any per-SKU patch render raises SkuFolderUnknownError. Renders crash before composite. Add 23 folder mappings to wordpress-theme/skyyrose-flagship/data/logo-registry.json. Each entry: "sku": "<slug>".
2 sg-011 and sg-012 have zero logo placements Original-Label Tee variants would render without any branding overlay. Add sku_logos.{sg-011,sg-012}.placements entries with logo_id, anchor, scale.

4 4 of 6 canonical angles < 50% covered (back, three-quarter, detail-1, detail-2) Multi-angle render outputs degrade to front-only. Detail-shot SSIM threshold (0.92) cannot enforce. Capture additional angles per SKU; uploader needs angle parameter (today is single-endpoint front).
5 Uploader UI writes front.jpg only — no angle picker Cannot upload back / three-quarter / detail / reference angles through the current FastAPI surface. Extend scripts/golden_uploader.py: add angle form field, route to golden_path(sku, angle=<...>). The helper already supports it.
6 scikit-image not installed in .venv Visual-regression SSIM degrades to passthrough (ssim_score=1.0). One test asserts this, fails. pip install scikit-image inside .venv.

5. SaaS-readiness notes

Per founder direction: build the SkyyRose imagery system as if it were the SaaS product. Multi-tenant later — single-brand quality first. The audit checks above are the SkyyRose surface; the patterns below are what makes them shippable to other brands without rework.

SaaS dimensionSkyyRose stateWhat scales / what's brand-locked
Catalog source CSV + per-SKU dossier (canonical 2026-05-27 lock). Scales. Both formats are tenant-isolatable — one CSV + one dossier dir per tenant.
Path resolution skyyrose/core/paths.py with golden_path() + wp_product_path() helpers (Phase 1+2 of this session). Mostly scales. The two helpers are pure functions; the constants hardcode the WordPress theme path. Multi-tenant version replaces constants with get_tenant_paths(tenant_id).
Golden ingestion Local FastAPI uploader on :8765. Brand-locked. Multi-tenant version needs auth, per-tenant write isolation, and S3/R2 backend instead of local filesystem.
Render pipeline skyyrose.elite_studio CLI + async worker + DLQ. Scales. Queue + DLQ pattern is tenant-agnostic. Per-tenant config injection via TrellisConfig.from_env()-style.
Brand canon enforcement Per-SKU dossier markdown is the single source. Hard-fail on missing dossier (DossierMissingError). Scales. Each tenant gets its own dossier set + canon validator.
Logo registry Per-brand logo-registry.json. Today: 14 logos, 9 folder mappings, 32 SKU placements. Scales. Schema is brand-agnostic; multi-tenant just multiplies the file count.
Compositor stages Seven stages: matte → prompt → relight → composite → cleanup → shadows → QA. Stage D defaults to deterministic rasterize (Phase 2 spec, this session). Scales. Every paid-API call gates on STOP-AND-SHOW + cost preview, which is multi-tenant friendly.
3D generation Modal TRELLIS (A10, $0.59/hr) + HF Space fallback. Phase A5 committed today (a00c871b9). Mostly scales. Modal namespace is workspace-scoped; multi-tenant requires per-tenant workspace or namespacing inside the function call.

6. Recommended next actions (before render)

  1. Extend uploader to accept angle. Add an angle form field on the FastAPI POST. The path helper already takes angle; only the FastAPI route + HTML form need the new field. Unblocks reference + back + three-quarter + detail uploads through the same UI.
  2. Populate the 7 missing reference.jpg baselines. Required for visual-regression SSIM scoring.
  3. Backfill sku_folders for the 23 unmapped SKUs. One JSON edit unblocks per-SKU patch placement across Love Hurts, Signature, and Kids Capsule entirely. Without it those SKUs cannot render any patch through LogoRegistry.image_path().
  4. Add logo placements for sg-011 and sg-012. Otherwise Original-Label Tee variants render unbranded.
  5. Install scikit-image in .venv. Visual-regression suite gets full SSIM coverage; the skipped test in test_visual_regression_multi_angle.py turns green.
  6. Capture back + detail angles per Black Rose first wave (br-001..br-006). The collection most likely to ship first; today only br-007 has full detail-1 coverage.
  7. THEN render. python -m skyyrose.elite_studio produce --sku br-007 first (the only SKU with 5/6 angle coverage + full sku_folders mapping). br-007 is the canary for the multi-angle pipeline.

Items 1–6 are pre-render. Item 7 is the actual render trigger. The current top-of-queue is br-007 because it is the single best-covered SKU; everything else needs at least one of the above gaps closed first.
