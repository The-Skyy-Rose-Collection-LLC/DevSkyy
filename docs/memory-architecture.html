DevSkyy Memory Architecture

Source of truth for how memory/context is stored, who writes it, and how it stays in sync. Established 2026-06-12. Pairs with .wolf/OPENWOLF.md (the OpenWolf protocol) and the auto-memory protocol in the global CLAUDE.md.

Core principle: every memory artifact has exactly one scope (repo-shared, machine-personal, or ephemeral) and one owner. Do not write the same fact into multiple systems — pick the scope. Derived/ephemeral artifacts are git-ignored and never hand-authored.

2. Overlap resolution (the alignment rule)

"Learnings / gotchas / preferences" historically lived in three places at once — cerebrum, auto-memory, and claude-mem. From now on, route by scope:

If the fact is…Write it toWhy
A repo-scoped convention or gotcha (true for anyone working this codebase).wolf/cerebrum.mdTravels with the repo; every clone/teammate gets it.
A personal, cross-project fact or preferenceAuto-memory fact fileYours across all projects; not repo-bound.
Raw "what happened in this session" historynothing — claude-mem captures itAuto-ingested. Query it; don't author it.
An active task / worklist itemtasks/todo.mdSingle worklist, survives session boundaries.
A bug + its fix.wolf/buglog.jsonSingle ledger, dedup by id.

3. Synchronization config (2026-06-12)

The problem: claude-mem's per-folder injection wrote <claude-mem-context> blocks into tracked CLAUDE.md files (342 of them). Because a background worker mutates them on a timer, any multi-step git op that resets the index (merge --abort, rebase, reset, branch checkout) raced the worker and failed with Entry 'X/CLAUDE.md' not uptodate.

The fix — redirect, don't disable (keep gathering info, stop the churn):

Setting (~/.claude-mem/settings.json)ValueEffect
CLAUDE_MEM_FOLDER_CLAUDEMD_ENABLEDtruePer-folder context injection stays ON (info still gathered).
CLAUDE_MEM_FOLDER_USE_LOCAL_MDtrueWorker writes to CLAUDE.local.md instead of CLAUDE.md (worker logic: USE_LOCAL_MD==="true" ? "CLAUDE.local.md" : "CLAUDE.md").
.gitignoreCLAUDE.local.mdIgnored at any depth — Claude Code still loads it as local context; git never sees it.

After changing settings, the worker must be restarted to pick up the new config: node <plugin>/scripts/bun-runner.js <plugin>/scripts/worker-service.cjs restart.

One-time cleanup (2026-06-12): the 342 pre-existing tracked CLAUDE.md with stale blocks were cleaned — 254 pure-stub files (block only, zero content) deleted; 88 content-bearing files had the block stripped, real content kept. The root CLAUDE.md and .wolf/cerebrum.md never carried blocks.
