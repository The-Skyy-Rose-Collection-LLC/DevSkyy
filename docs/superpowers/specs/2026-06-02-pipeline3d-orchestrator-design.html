Unified 3D Pipeline Orchestrator

Tripo3D/Meshy pipeline-orchestration-workflow clone · DevSkyy · 2026-06-02 · Status: APPROVED · Layer 2 (orchestration only)

A provider-agnostic staged 3D pipeline that mirrors how Tripo3D and Meshy stage async jobs (image-to-3D → texture/PBR → remesh → export). New package; existing provider clients become adapters behind ports. Engine strategy: provider router with fashion-tuned stages. Surfaces: REST API + webhook server + CLI. Export at launch: GLB only.

3. Execution spine — synchronous-within-stage

Each stage calls a blocking adapter that polls its provider to completion and returns an Artifact. Job-level concurrency comes from the existing Redis worker queue (many jobs in flight), not intra-job event advancement. This matches what the clients already do (TripoAssetAgent SDK polls internally, MeshyClient._wait_for_task, TrellisProvider retry loop). There is exactly one control model — no event-driven DAG advancement (that would race the poller).

Job(image, stages=[image_to_3d, texture, remesh, export], formats=[glb], budget)
  -> preflight.resolve(sku)          # canonical source image + hallucination guard
  -> estimator.estimate(job)         # ONE total cost, credits->USD
  -> forensics.confirm_prompt(total) # STOP-AND-SHOW once per job (paid dispatch only)
  -> executor.run(job):
       for stage in ordered_stages(job.stages):        # topological by depends-on
         if store.has(input_hash, stage, params):       # idempotent resume
             ctx.last_artifact = store.get(...).artifact ; continue
         adapter = router.pick(stage)                    # capability + priority + availability
         budget.ensure_within_budget(stage_cost, stage) # raises BudgetExceededError BEFORE paid call
         with telemetry.stage(stage_name=stage, ...) as t:
             result = adapter.run(ctx)                   # blocking: submit -> poller.wait -> download
         budget.spend(stage_cost, stage)
         store.put(input_hash, stage, params, result)
         ctx.last_artifact = result.artifact             # chain to next stage
  -> PipelineResult(artifacts, costs, timings, manifest_path)
Cross-provider chaining — the rule that makes the router real. Tripo chains by original_model_task_id, Meshy by input_task_id — but a Meshy task_id is meaningless to Tripo. So the unified Artifact carries both:
  • provider + task_id — cheap same-provider chaining
  • model_url / local path — cross-provider handoff (both platforms accept model_url as the alternative input)
Router rule: if next stage's chosen adapter == producing adapter, pass task_id; else hand off the downloadable model_url/path. This same seam is where a self-hosted engine plugs in later (Layer 3) with zero workflow change.

Artifact (the chaining handle)

@dataclass(frozen=True)
class Artifact:
    provider: str          # "tripo" | "meshy" | "trellis"
    task_id: str | None    # provider-native; None for self-host/local
    model_url: str | None  # remote download URL (may have TTL ~24h)
    path: Path | None      # local file once downloaded
    fmt: str               # "glb"
    polycount: int | None
    bytes: int | None
    meta: dict

4. Provider router + fallback

router.pick(stage) filters adapters by capabilities() (supports this stage?), orders by configured priority, skips unavailable (missing key / circuit-open via ai_3d/resilience.py). On stage failure it tries the next capable adapter (artifact handoff). Default priority: Tripo → Meshy → TRELLIS (Tripo is the most complete client). Capability map:

7. Money + idempotency requirements (mandatory)

  • Whole-job estimate ONCE. estimator sums per-stage credits→USD for the requested stages and shows the total via a single confirm_prompt before the job starts — not four prompts per job. Per-stage credit costs sourced from provider docs (Meshy: image-to-3D 5–30, retexture 10, remesh 5; Tripo: ~$0.40 image + texture/convert surcharges).
  • Paid dispatch stays gated. WordPress auto-deploy standing-auth does NOT cover paid 3D APIs. Default dry-run; --go / explicit confirm required.
  • Stage-level idempotency. A job that dies at remesh must NOT re-bill image-to-3D + texture on retry. Persist stage results keyed by (input_hash, stage, params) via telemetry.hash_inputs() + IdempotencyCache; resumed jobs skip completed stages.
