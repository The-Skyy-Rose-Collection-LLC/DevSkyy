1. Goal

One shared evaluation system that guarantees output accuracy before it ships, serving two domains through one core: imagery (does a generated product photo 100%-replicate the real garment?) and copy (is marketing/product text brand- and catalog-accurate?). Two distinct "job-title" agents, one core, swappable domains — the founder's ports-and-adapters model.

The two agents (job titles — confirm/adjust)

Both agents are thin: a job-title identity + a DomainAdapter. All real machinery lives in the shared core. Adding a third domain later (e.g. 3D meshes) = one new adapter.

3. The DomainAdapter port ("same system, slightly different domains")

class DomainAdapter(Protocol):
    domain: str                       # "imagery" | "copy"

    def deterministic_checks(self, subject, ref) -> list[CheckFailure]: ...
        # free, un-hallucinatable. imagery: PNG decode/size/collage-band + identity;
        # copy: catalog-fact match + placeholder + banned-phrase + structure

    def build_judge_request(self, subject, ref) -> JudgeRequest: ...
        # imagery: vision messages (candidate image + reference images + dossier text);
        # copy: text messages (draft + catalog/canon context). Both force a CoT-first schema.

    def parse_verdict(self, judge_output, det_failures) -> Verdict: ...
        # imagery: 6 boolean gates → pass/fail + failure_tags;
        # copy: 0–5 dimension scores → weighted composite

    async def revise(self, ref, critique) -> Subject: ...
        # imagery: re-render via scripts/oai_render pipeline with corrections;
        # copy: re-invoke the content agent with critique-augmented task

    def load_ground_truth(self) -> list[LabeledExample]: ...
        # imagery: renders/oai/_review/review-state.json (81 founder labels);
        # copy: tests/fixtures/.../labels (built later)

Verdict is normalized across domains: passed: bool, score: float (0–1), gate_results: dict, failure_tags: tuple[str,...], reason: str, cost_usd: float, attempts: int, mode: "hard_gate"|"soft_signal", domain: str. Imagery fills gate_results with the 6 booleans; copy fills it with dimension scores.

5. Content Accuracy Evaluator domain #2, later

Domain adapter: evaluation/domains/content.py. This is the previously-approved text evaluator, re-homed onto the shared core. The plan already written (docs/superpowers/plans/2026-06-16-evaluator-agent.md) maps directly: deterministic catalog-fact gate, Claude text judge (0–5 criterion-separated, CoT, blind, anti-verbosity), bounded revise loop, calibration. No re-design needed; it becomes a DomainAdapter implementation instead of a standalone package. Build after imagery ships.

6. Shared Observer

evaluation/observer.py — one Observer, domain-tagged. Per evaluation it records: domain, subject id (SKU or content id), passed, failure_tags, score, cost_usd, duration, attempts. Computes pass-rate, flag-rate, judge cost, judge-vs-human agreement (against the ground-truth set). Alert+recommend only (never auto-acts).

  • Imagery persistence: today only failures persist (quarantine sidecars, renders/oai/_rejected/); successes vanish. Observer writes a per-render outcome record (JSONL run-log under renders/oai/_runs/<run_id>.jsonl) + a batch summary. Optionally also the orphaned AgentExecution table (agents/models.py:201) — but JSONL is the low-risk v1 (that table is unwired and PG-typed).
  • Cost accuracy: current judge_cost_usd is a fixed $0.0002 constant (OpenAI). The Observer + the STOP-AND-SHOW manifest must compute real Claude-vision judge cost (input image tokens + output) before any paid run. required
  • Cross-domain: the same Observer records content evaluations when domain #2 lands — one dashboard, two domains.

8. Mandatory pre-fixes (flagged by the completeness critic)

  1. silent-fallback bug dossier_loader.py parses reference_image + extra_references from frontmatter but drops them from the Dossier dataclass. The imagery adapter needs the ground-truth photo paths — add these fields (or load via references.build_references()). Without this, the judge silently gets no reference. Violates no-silent-fallback canon.
  2. cost Replace the fixed EST_JUDGE_COST_USD=$0.0002 with real Claude-vision cost in the manifest before any paid run.
  3. Do not read placement.md (auto-generated scaffolding, non-authoritative); resolve canon from CSV + dossier + logo-registry. Verify GOLDEN_DIR/{sku}/reference.jpg provenance before trusting it as ground truth.

10. Risks & open questions

  • "6/6 Claude" is unverified — no committed harness ever proved it. Phase 1 calibration produces the real number; if κ < 0.65 the judge ships as soft-signal (annotate, don't block) until tuned.
  • ANTHROPIC_API_KEY wiring — not present in scripts/oai_render/config.py. Confirm the secret-load path (gemini/.env / .env.secrets) in Phase 0.
  • Deleted renders — only 2 PNGs survive; calibration uses the 81 labels but the images for flagged/approved renders are mostly gone (13 evidence imgs remain). Open question: do we calibrate the judge against the 13 recoverable evidence images + re-render a small labeled set, or re-render the full 81 first? (Affects Phase-1 paid cost.)
  • 4 SKUs lack a reference photo (black-rose-crewneck, black-rose-joggers, original-label-tee-orchid/white) — imagery judge needs a defined fallback (techflat) or reduced gate for these.
  • Job titles — "Render Fidelity Evaluator" / "Content Accuracy Evaluator" are proposals; confirm or rename.
