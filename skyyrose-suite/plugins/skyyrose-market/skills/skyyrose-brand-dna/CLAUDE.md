# skyyrose-brand-dna — Skill Context

This skill is the **canon foundation** for all SkyyRose creative and marketing work.

## Reading order when invoked

1. `SKILL.md` — brand identity, founder story, collections, palette, voice, tagline
2. `brand-guardrails.md` — operational rules: The Five visual references, lockup rule,
   SKU-vs-name rule, STOP-AND-SHOW gates, audit discipline, canonical product source

Both files must be read together. `SKILL.md` is the "what SkyyRose is".
`brand-guardrails.md` is the "what agents must never do".

## What this skill gates

Any agent producing content for SkyyRose — copy, design direction, product descriptions,
marketing briefs, social content, theme templates, imagery — must pass every rule in both files
before output is considered valid.

Downstream skills that reference this skill:
- `skyyrose-content-engine` (social content hub)
- `skyyrose-product-copy` (PDP copy)
- `skyyrose-seo-commerce` (SEO + structured data)
- All `skyyrose-social-*` skills (inherit `brand-guardrails.md` rules)

## Non-negotiables (quick-reference)

- Tagline verbatim: `Luxury Grows from Concrete.` (period included)
- Collections: Black Rose / Love Hurts / Signature / Kids Capsule — never cross-attribute voices
- Visual refs: The Five (Kith, Oaklandish, Culture Kings, Fear of God, Palm Angels) — never EU luxury houses
- Products by NAME not SKU in any customer-facing output
- Hero collection names = lockup PNG assets, never type-rendered
- No urgency timers, no cross-sell / related products on PDP
- STOP-AND-SHOW before paid API calls, Klaviyo sends, WC writes, deploys
