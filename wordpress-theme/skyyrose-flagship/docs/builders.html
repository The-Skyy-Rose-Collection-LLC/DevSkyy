Page Builder Compatibility

SkyyRose ships with first-class support for multiple page builders. The theme detects the active builder at runtime and loads only the relevant integration layer.

Builder Detection

The helper function skyyrose_active_builder() returns the slug of the currently active builder:

$builder = skyyrose_active_builder();
// Returns: 'gutenberg' | 'elementor' | 'divi' | 'beaver-builder' | 'bricks'

This function checks for active builder plugins in priority order and falls back to 'gutenberg' (the WordPress default) when no third-party builder is detected.

Gutenberg (Block Editor)

Default The native WordPress block editor is fully supported out of the box.

  • Block Patterns — Pre-built patterns for hero sections, product grids, collection banners, and testimonial rows are registered automatically.
  • Site Editor — SkyyRose includes block theme templates editable via Appearance → Editor. Modify headers, footers, and archive layouts visually.
  • theme.json — All design tokens (colors, typography, spacing) are defined in theme.json and surfaced in the editor UI.
  • Custom blocks — The skyyrose/holographic-card and skyyrose/collection-hero blocks are registered for Gutenberg only.

Divi 5.x

Divi 5.x is supported with the following considerations:

  • The <model-viewer> web component is allow-listed so 3D product previews render inside Divi modules.
  • SkyyRose CSS custom properties (--skyyrose-*) are available in Divi's custom CSS fields.
  • Use the Code Module to embed Three.js holographic cards or GSAP scroll animations.

Beaver Builder 2.x

  • Template Directory — SkyyRose registers pre-built page templates in Beaver Builder's template directory (Templates → Saved Templates).
  • Templates include: Landing Page, Collection Archive, Product Showcase, and About Page.
  • Row and module styles inherit theme design tokens automatically.

Bricks 1.x

  • Bricks reads theme.json natively, so all SkyyRose design tokens (colors, fonts, spacing) appear in the Bricks editor without extra configuration.
  • Global classes using --skyyrose-* custom properties work in the Bricks style panel.
  • Dynamic data tags for WooCommerce product meta (care instructions, fabric content) are compatible.

Page Templates

Two layout templates are available for every supported builder:

TemplateSlugDescription
Canvastemplate-canvas.phpNo header, no footer. Full control for landing pages and custom funnels.
Full Widthtemplate-full-width.phpHeader and footer included, no sidebar, content spans the full viewport width.

Select the template from the Page Attributes panel (Gutenberg) or the builder's own template selector.

Best Practices

  • Only one builder plugin should be active at a time. skyyrose_active_builder() returns the first match.
  • Deactivate unused builders to keep front-end asset loading minimal.
  • When switching builders, re-save permalinks (Settings → Permalinks → Save) to flush rewrite rules.
